Review the clautorun daemon storage architecture in `plugins/clautorun/src/clautorun/session_manager.py`.

**Context**: clautorun is a Python asyncio daemon that serves hooks from multiple concurrent Claude Code and Gemini CLI sessions. It uses `shelve` (backed by dbm.ndbm on macOS, writing to a 50MB `.db.db` file) as persistent shared state, protected by 420 lines of custom `fcntl.flock()` locking (SessionLock, SessionBackendManager, SessionStateManager). 

**Current Bug**: The daemon gets permanently stuck at 98.5% CPU after handling `PostToolUse(ExitPlanMode)` — FDs for the lock file and shelve DB are leaked, the asyncio event loop blocks indefinitely because `handle_client()` calls `self.app.dispatch(ctx)` SYNCHRONOUSLY (no await). Tests timeout because no new connections can be served.
