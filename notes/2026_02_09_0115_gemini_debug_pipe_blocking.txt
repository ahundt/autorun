# Pipe Blocking Bug - Gemini Debug Request

## Problem Summary

User command is being blocked even though fix is implemented and tests pass:

**Blocked Command**: `gemini extensions list | grep -A 2 -B 2 clautorun || echo 'No clautorun found'`

**Error**: "Command blocked: grep" even though grep is in a pipe and should be allowed.

## What I Fixed

1. **Heredoc bug in command_detection.py**:
   - Added `_normalize_heredoc_delimiters()` function
   - Converts `<< 'EOF'` to `<< EOF` for bashlex compatibility
   - Tests pass: 20/20 (9 new + 11 existing)

2. **Root cause**:
   - Bashlex failed to parse heredocs with quoted delimiters
   - Fell back to shlex parser which treated heredoc LINES as commands
   - Line "pattern = grep" got added to all_potential set

3. **The fix works in tests**:
```python
# test_pipe_detection_debug.py output:
Command: gemini extensions list | grep -A 2 -B 2 clautorun || echo 'No clautorun found'
Step 1 - command_matches_pattern(cmd, 'grep'): True  ✅
Step 2 - _not_in_pipe(ctx): False  ✅ (correctly detected pipe)
RESULT: Command would be ALLOWED ✅
```
